Utilisation de Git et Github

Site ref : https://doc.ubuntu-fr.org/git

Git est un logiciel de gestion de versions décentralisé.

GitHub (exploité sous le nom de GitHub, Inc.) est un service web d'hébergement et
de gestion de développement de logiciels, utilisant le logiciel de gestion de versions Git.

Gérer les dépots

Initialiser un depotgit init
Récupérer un depot deja existantgit clone git//depot
ou
git clone https://depot

État du dépot

Comparer 2 versionsgit diff
Savoir tout ce qui n’a pas encore été validé git status
Lister les commits effectués dans le dépotgit log

Gestion des fichiers

Versionner un fichiergit add nom_fichier_ou_dossier
Ajouter tout le contenu (fichier ou dossier) du dossiergit add --all *
Supprimer le fichier de l'ordinateur et du depot Gitgit rm nom_fichier
Déplacer le fichiergit mv nom_fichier nouvelle_destination

Gestion des commits

Mettre a jour votre depot localgit pull
Créer un commit avec un commentairegit commit -m "com"
Créer un commit contenant fichier et fichier2git commit fichier1 fichier2
Créer un nouveau commit contenant tout les changements effectués sur les fichiers
(git add n'est pas nécessaire avant cimmit -a)		git commit -a
Envoyer le commit dans la branche principale"master" du dépot "origin"git push origin master

Commandes d'annulation

Annule les changements effectués depuis le dernier commit. git reset –hard HEAD
Supprime le dernier commit. Cette action peut être répétée autant de fois que vous le désirez. git reset –hard HEAD^
Restaure le dépôt tel qu'il était lors du commit spécifié. git revert commit
Pour que cette commande fonctionne, il faut que toutes les modifications soient commitées

Utilisation avancée

Gestion des branches

Créer une nouvelle branche nommée "test"git branch test
Lister les branches présentes dans le dépotgit branch
Changer de branchegit checkout nom_branche
Fusionner la branche courante avec la branche nommée nom_branchegit merge nom_branche
Abandonner la fusiongit reset --hard HEAD

Récupération des changements
